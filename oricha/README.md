# Oricha 降車アラート

降車駅を登録しておくと、近づいたときに **音・画面フラッシュ・バイブ（対応端末）** で知らせる乗り過ごし防止 PWA。降車駅は何個でも登録できる。

## ファイル構成
```
oricha/
├─ index.html      … アプリ本体（UI + GPS監視 + Wake Lock + アラート + デモ走行）
├─ stations.js     … 駅データ（全国8766駅、駅データ.jp由来）
├─ scripts/
│  └─ convert-stations.js … 駅データ.jp の station/line CSV から stations.js を再生成するスクリプト
├─ sw.js           … Service Worker（オフライン起動 / インストール用）
├─ manifest.json   … PWA設定
└─ icons/          … アイコン
```

## 動かし方

### ① 手元（PC）で確認
GPS・Wake Lock・Service Worker は **HTTPS か localhost でしか動かない**。`file://` で直接開くと動かないので、簡易サーバーを立てる:
```bash
cd oricha
python3 -m http.server 8000
# → http://localhost:8000 を開く
```
電車に乗らなくても、駅を1つ登録して **「デモ走行」** を押せば、仮想GPSで接近〜アラートまで体験できる。

### ② スマホ実機で確認（推奨）
実機の GPS を使うには HTTPS が必要。一番ラクなのは無料ホスティングに置く:
- **Vercel** か **Netlify** にこのフォルダをデプロイ → 発行された https URL をスマホで開く
- スマホで「ホーム画面に追加」するとアプリっぽく起動する（standalone）

## 使い方
1. 検索ボックスに駅名（漢字 or ひらがな）を入れて候補から追加。何個でも登録可。
2. 「予告」「本アラート」の距離を選ぶ（初期値：1.5km手前で黄色 / 500m手前で発報）。
3. 「画面を消さない（Wake Lock）」を ON にする（電車に乗っている間だけ ON 推奨）。
4. 「監視開始」。最寄りの登録駅までの距離が大きく表示され、近づくと色が変わってアラートが鳴る。

## 駅データを更新する
`stations.js` は駅データ.jp（https://www.ekidata.jp/ ）の station/line CSV から `scripts/convert-stations.js` で生成している。最新版に更新する場合:
1. 駅データ.jpで会員登録し、最新の `station*.csv` と `line*.csv` をダウンロード
2. `node scripts/convert-stations.js <station.csv> <line.csv> stations.js` を実行

注意:
- 無料版CSVには読み仮名（ひらがな）列が無いため `yomi` は空文字になる。検索は漢字の駅名のみで行われる。
- `e_status` が `0`（現存）の駅のみを対象とし、廃止駅は除外している。
- 検索側（index.html）は駅名・よみを小文字キャッシュした配列を先頭から走査し、20件見つかったら即終了する実装になっており、8000駅超でも軽快に動く。

## 既知の制約（重要）
- **完全なバックグラウンド追跡はできない。** iOS も Android も、ブラウザ/PWA にバックグラウンドGPSやジオフェンスを許可していない。だから「画面を点けてアプリを前面に出したまま」が前提（Wake Lock で画面が消えないようにしている）。完全バックグラウンドが必要になったら、その時は Flutter / React Native などネイティブへ。
- **iPhone（Safari）はバイブ非対応。** iOS では音と画面フラッシュで知らせる。Android は加えてバイブも鳴る。
- 地下・トンネルでは GPS が途切れることがある（電車では一時的に距離が更新されない場合あり）。
